# Binaa ERP - Architecture Overhaul Changelog

## الإصدار: التحديث المعماري الشامل (Architecture Overhaul v2.0)

---

## ملخص التحديثات (Summary of Changes)

تم إجراء تحديث معماري شامل لجعل **دليل الحسابات المحرك الحقيقي للنظام** بالكامل،
محاكياً أنظمة SAP وOracle NetSuite وDynamics 365.

---

## 1. المعادلة المعمارية الكاملة (Ultimate Architecture Flow)

```
دليل الحسابات (Chart of Accounts)
       ↓
الأدوار الوظيفية (Account Roles)
       ↓
محرك الربط المالي (Financial Mapping Engine)
       ↓
العمليات التجارية (Business Operations)
       ↓
القيود المحاسبية (Journal Entries)
       ↓
دفتر الأستاذ العام (General Ledger)
       ↓
القوائم المالية (Financial Statements)
```

---

## 2. الملفات الجديدة المضافة (New Files Added)

### محرك الربط المالي (Financial Mapping Engine)
- **`src/lib/financial-mapping-engine.ts`** (698 سطر)
  - 24 نوع عملية تجارية معرفة بالكامل
  - دالة `resolveOperationAccounts()` - تحل نوع العملية إلى حسابات فعلية
  - دالة `resolveRoleToAccounts()` - تحل الدور إلى حسابات (أب → أبناء)
  - دالة `seedFinancialMappings()` - تهيئة الربط الافتراضي
  - دالة `getDefaultAccountForOperation()` - الحساب الافتراضي للعمليات التلقائية

### فحص السلامة المحاسبية (Accounting Health Check)
- **`src/lib/accounting-health-check.ts`** (457 سطر)
  - 7 فحوصات شاملة على النظام المحاسبي
  - فحص الأدوار بدون ربط (حرج)
  - فحص الحسابات المعطلة في الربط (تحذير)
  - فحص الأدوار بحسابات أب فقط (تحذير)
  - فحص الأدوار بحسابات متعددة (معلومة)
  - فحص القيود بحسابات معطلة (حرج)
  - فحص العمليات بدون ربط (تحذير)
  - فحص الأدوار على حسابات أب (معلومة)
  - حساب نتيجة مئوية مع مرجحة الخطورة

### أثر الحسابات (Account Impact)
- **`src/lib/account-impact.ts`** (395 سطر)
  - تحليل شامل لاستخدام كل حساب
  - معلومات الحساب الأساسية (أب، أبناء، دور)
  - العمليات المستخدمة (مدين/دائن/كلاهما)
  - إحصائيات الاستخدام (عدد القيود، الأرصدة، آخر استخدام)
  - مراجع المستندات (فواتير، تحصيلات، مدفوعات)
  - حظر الإلغاء مع أسباب مفصلة

### أدوار الحسابات (Account Roles)
- **`src/lib/account-roles.ts`** (614 سطر)
  - 31 دور وظيفي معرف
  - `getDefaultAccountByRole()` - الحساب الافتراضي للدور
  - `getChildAccountsByRole()` - الحسابات الفرعية للدور
  - `getAccountsByRole()` / `getAccountsByRoles()` - استعلام بالأدوار
  - `requireAccountByRole()` - يتطلب حساب مع خطأ عربي وصفي
  - `getRoleAccountMapping()` - خريطة الربط الكاملة

## 3. الملفات المُحدّثة (Updated Files)

### محرك المحاسبة (Accounting Engine)
- **`src/lib/accounting/engine.ts`** (~2061 سطر)

  - أضيفت دالة `reverseEntry()` عامة لعكس القيود

  - حقل `parentCode` للعلاقة الأب-الابن
  - حقل `isActive` للحذف الناعم
  - حقل `allowPosting` لتمييز الحسابات القابلة للترحيل
  - نموذج `FinancialMapping` (محرك الربط المالي)
  - نموذج `AccountingHealthCheck` (نتائج الفحص)
  - حقل `expenseAccountId` على Employee (حساب مصروف الراتب)
  - حقل `assetAccountId` على Equipment (حساب الأصل)
  - استبدال حقول معرفات الحسابات الفردية بـ `journalEntryId`

## 4. نقاط API الجديدة (New API Endpoints)

  - تحليل أثر حساب معين
- **`/api/accounts/role-mapping`** (GET/PUT/POST)
  - ربط الأدوار بالحسابات
- **`/api/accounts/statement`** (GET)
  - كشف حساب لأي حساب
- **`/api/accounts/by-role`** (GET)
  - استعلام الحسابات حسب الدور
- **`/api/journal-entries/[id]/reverse`** (POST)
  - عكس قيد محاسبي

## 8. الميزات الست المنفذة (6 Implemented Features)

### 1. نمط الدور ← مجموعة الحساب ← الحساب ✅
- الدالة `resolveRoleToAccounts()` تحل الأدوار إلى حسابات
- الحسابات التجميعية تُستبدل بأبنائها تلقائياً
- مثال: BANK → 1120 (أب) → [1121 الراجحي، 1122 الأهلي، 1123 الإنماء]

### 2. ربط ADMIN_EXPENSE ✅
- الدور مربوط بالحساب الأب 8100
- 7 حسابات فرعية (8120-8170) قابلة للاختيار

### 3. شاشة أثر الحسابات ✅
- تعرض العمليات والمستندات والإحصائيات
- حظر التعطيل مع أسباب مفصلة

### 4. الحذف الناعم ✅
- لا يمكن حذف حساب مستخدم في قيود
- التعطيل فقط مع فحص الأثر أولاً
- `deactivateAccount()` مع حظر ذكي

### 5. محرك الربط المالي ✅
- 24 نوع عملية معرفة بالكامل
- طبقة منطق محاسبي مستقلة
- تغيير الحساب ينتشر تلقائياً

### 6. فحص السلامة المحاسبي ✅
- 7 فحوصات يومية
- مؤشر لوني على لوحة القيادة (🟢🟡🔴)
- نتيجة مئوية مع مرجحة الخطورة
