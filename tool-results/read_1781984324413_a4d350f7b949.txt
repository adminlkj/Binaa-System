    75→- Inserted after `initializeChartOfAccounts()` (before `createJournalEntry`) at line ~371
    76→- Function signature: `reverseEntry(journalEntryId: string, tx: PrismaTransaction)` — tx is REQUIRED (no fallback to db)
    77→- Validation checks:
    78→  - Entry must exist (throws Arabic error if not found)
    79→  - Entry must be POSTED status (throws if DRAFT or CANCELLED)
    80→  - Entry must not already be a reversal (`isReversal = false`)
    81→  - Entry must not already have been reversed (checks for existing entry with `reversedEntryId = originalEntryId`)
    82→- Creates a new reversal entry with:
    83→  - Same date as original
    84→  - Description prefixed with "عكس - "
    85→  - `status: 'POSTED'`
    86→  - Same `sourceType` and `sourceId` as original
    87→  - `isReversal: true`
    88→  - `reversedEntryId` pointing to original entry
    89→  - `isSystem: true`
    90→  - Lines with flipped debit/credit (each line's description prefixed with "عكس - ")
    91→- Updates the original entry `status` to `CANCELLED`
    92→- Returns the reversal entry with `include: { lines: true }`
    93→- Uses same entry number generation pattern as `auto-journal.ts`
    94→
    95→**2. Created journal entries reversal API route at `/src/app/api/journal-entries/[id]/reverse/route.ts`:**
    96→- POST handler accepting `{ id }` param
    97→- Wraps everything in `prisma.$transaction()`
    98→- Calls `reverseEntry(id, tx)` within the transaction
    99→- Returns the reversal entry on success
   100→- Returns `{ error: message }` with status 400 on failure (Arabic error messages from engine)
   101→
   102→**3. Updated client-payments API route (`/src/app/api/client-payments/[id]/route.ts`):**
   103→- PATCH handler now supports reverse+recreate for POSTED payments:
   104→  1. Reverses the original journal entry via `reverseEntry()`

   131→Key design decisions:
   132→- `reverseEntry()` REQUIRES a transaction client — cannot be called standalone, enforcing atomicity
   133→- Original entry is CANCELLED (not deleted) — full audit trail preserved
