     3тЖТ     3тЖТ     3тЖТ## Color Philosophy
     4тЖТ     4тЖТ     4тЖТ
     5тЖТ     5тЖТ     5тЖТGLM uses a **mood-driven dynamic color system** instead of fixed named palettes. Colors are constructed from three dimensions:
     6тЖТ     6тЖТ     6тЖТ
     7тЖТ     7тЖТ     7тЖТ### Three Dimensions of Document Color
     8тЖТ     8тЖТ     8тЖТ
     9тЖТ     9тЖТ     9тЖТ| Dimension | Description | Range |
    10тЖТ    10тЖТ    10тЖТ|-----------|-------------|-------|
    11тЖТ    11тЖТ    11тЖТ| **Temperature** | Warm тЖФ Cool | Warm (consulting, education) тЖФ Cool (tech, medical) |
    12тЖТ    12тЖТ    12тЖТ| **Weight** | Light тЖФ Heavy | Light (resume, proposal) тЖФ Heavy (legal, academic) |
    13тЖТ    13тЖТ    13тЖТ| **Energy** | Calm тЖФ Active | Calm (official, contract) тЖФ Active (report, presentation) |
    14тЖТ    14тЖТ    14тЖТ
    15тЖТ    15тЖТ    15тЖТ### Color Token System
    16тЖТ    16тЖТ    16тЖТ
    17тЖТ    17тЖТ    17тЖТEvery document uses 5 color tokens. These are **computed** based on the document's mood, not selected from a fixed list.
    18тЖТ    18тЖТ    18тЖТ
    19тЖТ    19тЖТ    19тЖТ| Token | Role | Guidance |
    20тЖТ    20тЖТ    20тЖТ|-------|------|----------|
    21тЖТ    21тЖТ    21тЖТ| `primary` | Headings, cover title | Dark, authoritative. Derived from Temperature + Weight. |
    22тЖТ    22тЖТ    22тЖТ| `body` | Body text | Near-black with subtle warmth/coolness. Always high contrast. |
    23тЖТ    23тЖТ    23тЖТ| `secondary` | Captions, footnotes | Mid-tone gray. Legible but visually recessive. |
    24тЖТ    24тЖТ    24тЖТ| `accent` | Table headers, lines, links | The "personality" color. Reflects document Energy. |
    25тЖТ    25тЖТ    25тЖТ| `surface` | Table alternating rows, card backgrounds | Very light tint of accent or neutral. |
    26тЖТ    26тЖТ    26тЖТ

   157тЖТ   157тЖТ   157тЖТ### Visual Profile Color Guidance
   158тЖТ   158тЖТ   158тЖТ
   159тЖТ   159тЖТ   159тЖТFor **Profile B (Visual)** scenes тАФ resume, copywriting, and other non-formal documents тАФ prefer **warm neutral** tones aligned with the "Invisible Precision" design philosophy:
   160тЖТ   160тЖТ   160тЖТ
   161тЖТ   161тЖТ   161тЖТ- **Body text**: Use warm dark neutrals (`#37352F`, `#303030`, `#3A3430`) instead of cool blue-grays. This reduces eye strain.
   162тЖТ   162тЖТ   162тЖТ- **Surface/background**: Use warm near-white (`#F7F7F7`, `#FAFAF8`, `#FBF9F7`) instead of cool tints.
   163тЖТ   163тЖТ   163тЖТ- **Accent colors**: Use sparingly and only for functional differentiation (section headers, key metrics, links). 95% of the document should remain monochromatic (black, white, gray).
   164тЖТ   164тЖТ   164тЖТ- **Tables**: Prefer the **Zebra Stripe** style (see Table Styles ┬з2) тАФ hierarchy through background contrast with minimal borders. Fallback: Horizontal-Only.
   165тЖТ   165тЖТ   165тЖТ
   166тЖТ   166тЖТ   166тЖТThis does NOT apply to Profile A (Formal) scenes (report, academic, contract, official-doc, exam), which must retain pure black `"000000"` body text per regulatory standards.
   167тЖТ   167тЖТ   167тЖТ
   168тЖТ   168тЖТ   168тЖТ### Custom Color Generation
   169тЖТ   169тЖТ   169тЖТ
   170тЖТ   170тЖТ   170тЖТWhen the pre-defined recipes don't fit, construct colors using these rules:
   171тЖТ   171тЖТ   171тЖТ
   172тЖТ   172тЖТ   172тЖТ1. **primary**: Start from `hsl(hue, 25-40%, 10-18%)` тАФ dark, desaturated
   173тЖТ   173тЖТ   173тЖТ2. **body**: primary lightened 5-8% тАФ readable dark
   174тЖТ   174тЖТ   174тЖТ3. **secondary**: primary lightened 30-40% тАФ clearly subordinate
   175тЖТ   175тЖТ   175тЖТ4. **accent**: Choose a hue reflecting the domain, `hsl(domainHue, 30-50%, 45-55%)`
   176тЖТ   176тЖТ   176тЖТ5. **surface**: accent desaturated to 5-10%, lightened to 96-98%
   177тЖТ   177тЖТ   177тЖТ
   178тЖТ   178тЖТ   178тЖТ**Contrast check**: body text on white must achieve WCAG AA (тЙе4.5:1). All recipes above pass this.

   328тЖТ   328тЖТ   328тЖТ
   329тЖТ   329тЖТ   329тЖТEach palette in `coverPalettes` provides a `table` object with pre-computed colors for all 3 table styles:
   330тЖТ   330тЖТ   330тЖТ
   331тЖТ   331тЖТ   331тЖТ| Token | Used in | Description |
   332тЖТ   332тЖТ   332тЖТ|-------|---------|-------------|
   333тЖТ   333тЖТ   333тЖТ| `table.headerBg` | Zebra Stripe, Horizontal-Only | Table header background color |
   334тЖТ   334тЖТ   334тЖТ| `table.headerText` | Zebra Stripe, Horizontal-Only | Table header text color (must pass WCAG AA contrast) |
   335тЖТ   335тЖТ   335тЖТ| `table.accentLine` | Three-Line | Top/bottom/header-bottom line color |
   336тЖТ   336тЖТ   336тЖТ| `table.innerLine` | Horizontal-Only | Inner horizontal separator line color |
   337тЖТ   337тЖТ   337тЖТ| `table.surface` | Zebra Stripe | Alternating row background (light tint) |
   338тЖТ   338тЖТ   338тЖТ
   339тЖТ   339тЖТ   339тЖТ**Usage:**
   340тЖТ   340тЖТ   340тЖТ```js

   342тЖТ   342тЖТ   342тЖТconst t = palette.table;
   343тЖТ   343тЖТ   343тЖТ// Three-Line: use t.accentLine for border colors
   344тЖТ   344тЖТ   344тЖТ// Zebra: use t.headerBg, t.headerText, t.surface
   345тЖТ   345тЖТ   345тЖТ// Horizontal: use t.headerBg, t.headerText, t.innerLine
   346тЖТ   346тЖТ   346тЖТ```
   347тЖТ   347тЖТ   347тЖТ
   348тЖТ   348тЖТ   348тЖТ**тЪая╕П High-saturation accent override**: For DM-1, FG-1, and SN-2, the table colors are intentionally darkened/desaturated relative to the cover accent.    353тЖТ   353тЖТ   353тЖТ## Cover Page Design System
   354тЖТ   354тЖТ   354тЖТ
   355тЖТ   355тЖТ   355тЖТ### Design Philosophy
   356тЖТ   356тЖТ   356тЖТ
   357тЖТ   357тЖТ   357тЖТCovers use **7 validated layout recipes** + **parameterized variants** instead of free combination. Each recipe's background, layout, and decoration are visually verified. Differentiation comes from **palette ├Ч font size ├Ч content variation**.
   358тЖТ   358тЖТ   358тЖТ
   359тЖТ   359тЖТ   359тЖТ**Architecture principle:** All recipes use a **single 16838 outer wrapper table** (one row, exact height). Recipes R1/R2/R3 use **ZERO nested tables** тАФ all decoration is achieved via **paragraph borders** (left/right/top/bottom). This ensures maximum cross-engine stability (MS Office + WPS).

   648тЖТ   648тЖТ   648тЖТ
   649тЖТ   649тЖТ   649тЖТ1. Cover section page margin is 0; total content height тЙд 15638 twips (1200 twips safety margin for cross-engine compatibility тАФ MS Office renders large fonts taller than calculated).
   650тЖТ   650тЖТ   650тЖТ2. Color block Table `height` must use `rule: "exact"` (never `"atLeast"`).
   651тЖТ   651тЖТ   651тЖТ3. Each recipe code includes height budget annotations тАФ verify during generation.
   652тЖТ   652тЖТ   652тЖТ4. **`verticalAlign` must always be `"top"`**. Never use `"center"` or `"bottom"` in exact-height rows тАФ content will be clipped or overflow. Use `spacing.before` on the first paragraph for vertical positioning.
   653тЖТ   653тЖТ   653тЖТ5. **Title font size MUST be dynamically calculated via `calcTitleLayout()`** (see below). Never hardcode font sizes above 40pt for cover titles. Every recipe MUST call `calcTitleLayout()` before building title paragraphs.
   654тЖТ   654тЖТ   654тЖТ6. **Never use `margins.top`/`margins.bottom` for vertical positioning inside exact-height cells**: Cell margins reduce available height unpredictably across MS Office and WPS.    655тЖТ   655тЖТ   655тЖТ7. **Dynamic spacing is mandatory**: Use `calcCoverSpacing()` to compute `spacing.before` values dynamically based on content element count and title line count. Never use fixed large spacing values (e.g., `before: 4500`) that assume a specific title length.
   656тЖТ   656тЖТ   656тЖТ8. **Cover must be a single-page section**: The cover section must produce exactly ONE page. If content overflows to a second page, it means the height budget is violated. Common overflow causes and fixes:
   657тЖТ   657тЖТ   657тЖТ   - Title font too large тЖТ use `calcTitleLayout()` to auto-reduce
   658тЖТ   658тЖТ   658тЖТ   - Too many meta lines тЖТ reduce font size or remove less important lines
   659тЖТ   659тЖТ   659тЖТ   - Fixed `spacing.before` values too large тЖТ use `calcCoverSpacing()` for dynamic values
   660тЖТ   660тЖТ   660тЖТ   - Subtitle + English label + meta lines combined exceed budget тЖТ calculate total and reduce spacing
   661тЖТ   661тЖТ   661тЖТ9. **Cover wrapper table MUST use explicit `allNoBorders`**: The outer 16838 wrapper table and ALL nested tables inside the cover MUST set borders to NONE explicitly.    690тЖТ   690тЖТ   690тЖТ```
   691тЖТ   691тЖТ   691тЖТ9. **Post-generation overflow check (mandatory)**: After building cover children, estimate total height:
   692тЖТ   692тЖТ   692тЖТ   ```js
   693тЖТ   693тЖТ   693тЖТ   function estimateCoverHeight(elements) {
   694тЖТ   694тЖТ   694тЖТ     let total = 0;
   695тЖТ   695тЖТ   695тЖТ     for (const el of elements) {
   696тЖТ   696тЖТ   696тЖТ       if (el instanceof Table) {
   697тЖТ   697тЖТ   697тЖТ         // Sum row heights (exact rows) or estimate 400 twips per row
   698тЖТ   698тЖТ   698тЖТ         const rows = el.root?.[0]?.rows || [];
   699тЖТ   699тЖТ   699тЖТ         for (const row of rows) {
   700тЖТ   700тЖТ   700тЖТ           total += row.height?.value || 400;
   701тЖТ   701тЖТ   701тЖТ         }
   702тЖТ   702тЖТ   702тЖТ       } else if (el instanceof Paragraph) {
   703тЖТ   703тЖТ   703тЖТ         const fontSize = el.root?.[0]?.size || 24; // half-pts
   704тЖТ   704тЖТ   704тЖТ         // тШЕ Use pt * 11.5 (= half-pt * 10 * 1.15) for accurate single-spacing estimate
   705тЖТ   705тЖТ   705тЖТ         const lineHeight = Math.max(fontSize * 11.5, 276); // min 276 (default 12pt)
   706тЖТ   706тЖТ   706тЖТ         const spacingBefore = el.spacing?.before || 0;
   707тЖТ   707тЖТ   707тЖТ         const spacingAfter = el.spacing?.after || 0;
   708тЖТ   708тЖТ   708тЖТ         total += lineHeight + spacingBefore + spacingAfter;
   709тЖТ   709тЖТ   709тЖТ       }
   710тЖТ   710тЖТ   710тЖТ     }
   711тЖТ   711тЖТ   711тЖТ     return total;
   712тЖТ   712тЖТ   712тЖТ   }
   713тЖТ   713тЖТ   713тЖТ   // тШЕ Target: estimateCoverHeight(coverChildren) < 15638 (16838 - 1200 safety)
   714тЖТ   714тЖТ   714тЖТ   ```

   718тЖТ   718тЖТ   718тЖТ### тЪая╕П Cover Title Layout тАФ calcTitleLayout() (Mandatory for ALL Recipes)
   719тЖТ   719тЖТ   719тЖТ
   720тЖТ   720тЖТ   720тЖТ**Every cover recipe MUST use `calcTitleLayout()` to determine title font size and line breaks.** Hardcoding font sizes or passing the full title as a single TextRun is FORBIDDEN.
   721тЖТ   721тЖТ   721тЖТ
   722тЖТ   722тЖТ   722тЖТ**Every paragraph with font size > body text MUST set explicit line spacing** to prevent top clipping:
   723тЖТ   723тЖТ   723тЖТ```js
   724тЖТ   724тЖТ   724тЖТ// тШЕ MANDATORY: prevent inherited small line spacing from clipping large fonts
   725тЖТ   725тЖТ   725тЖТspacing: { line: Math.ceil(titlePt * 23), lineRule: "atLeast", after: 100 }
   726тЖТ   726тЖТ   726тЖТ// Example: 36pt тЖТ line: 828; 44pt тЖТ line: 1012
   727тЖТ   727тЖТ   727тЖТ```
   728тЖТ   728тЖТ   728тЖТWithout this, the paragraph inherits body text line spacing (e.g., 560tw), which is shorter than the font height тЖТ top of characters gets clipped.
   729тЖТ   729тЖТ   729тЖТ
   730тЖТ   730тЖТ   730тЖТ```js
   731тЖТ   731тЖТ   731тЖТ/**
   732тЖТ   732тЖТ   732тЖТ * Calculate safe font size and smart line breaks for cover titles.
   733тЖТ   733тЖТ   733тЖТ * MUST be called by every recipe before building title paragraphs.
   734тЖТ   734тЖТ   734тЖТ *
   735тЖТ   735тЖТ   735тЖТ * @param {string}  title          Full title string
   736тЖТ   736тЖТ   736тЖТ * @param {number}  maxWidthTwips  Available width for title text (twips, after subtracting margins/padding)
   737тЖТ   737тЖТ   737тЖТ * @param {number}  preferredPt    Desired max font size in pt (default 40)
   738тЖТ   738тЖТ   738тЖТ * @param {number}  minPt          Minimum allowed font size in pt (default 24)
   739тЖТ   739тЖТ   739тЖТ * @returns {{ titlePt: number, titleLines: string[] }}
   740тЖТ   740тЖТ   740тЖТ */
   741тЖТ   741тЖТ   741тЖТfunction calcTitleLayout(title, maxWidthTwips, preferredPt = 40, minPt = 24) {
   742тЖТ   742тЖТ   742тЖТ  // Each CJK character width тЙИ pt ├Ч 20 twips
   743тЖТ   743тЖТ   743тЖТ  const charWidth = (pt) => pt * 20;
   744тЖТ   744тЖТ   744тЖТ  const charsPerLine = (pt) => Math.floor(maxWidthTwips / charWidth(pt));
   745тЖТ   745тЖТ   745тЖТ
   746тЖТ   746тЖТ   746тЖТ  // Try from preferredPt downward until title fits in тЙд 3 lines
   747тЖТ   747тЖТ   747тЖТ  let titlePt = preferredPt;
   748тЖТ   748тЖТ   748тЖТ  let lines;
   749тЖТ   749тЖТ   749тЖТ  while (titlePt >= minPt) {
   750тЖТ   750тЖТ   750тЖТ    const cpl = charsPerLine(titlePt);
   751тЖТ   751тЖТ   751тЖТ    if (cpl < 2) { titlePt -= 2; continue; }
   752тЖТ   752тЖТ   752тЖТ    lines = splitTitleLines(title, cpl);
   753тЖТ   753тЖТ   753тЖТ    if (lines.length <= 3) break;
   754тЖТ   754тЖТ   754тЖТ    titlePt -= 2;
   755тЖТ   755тЖТ   755тЖТ  }
   756тЖТ   756тЖТ   756тЖТ
   757тЖТ   757тЖТ   757тЖТ  // If still > 3 lines at minPt, force 3 lines
   758тЖТ   758тЖТ   758тЖТ  if (!lines || lines.length > 3) {
   759тЖТ   759тЖТ   759тЖТ    const cpl = charsPerLine(minPt);
   760тЖТ   760тЖТ   760тЖТ    lines = splitTitleLines(title, cpl);
   761тЖТ   761тЖТ   761тЖТ    titlePt = minPt;
   762тЖТ   762тЖТ   762тЖТ  }
   763тЖТ   763тЖТ   763тЖТ
   764тЖТ   764тЖТ   764тЖТ  return { titlePt, titleLines: lines };
   765тЖТ   765тЖТ   765тЖТ}
   766тЖТ   766тЖТ   766тЖТ
   767тЖТ   767тЖТ   767тЖТ/**
   768тЖТ   768тЖТ   768тЖТ * Smart Chinese title line-breaking тАФ breaks at semantic boundaries, never mid-word.
   769тЖТ   769тЖТ   769тЖТ *
   770тЖТ   770тЖТ   770тЖТ * Rules:
   771тЖТ   771тЖТ   771тЖТ * 1. Prefer breaking after particles, punctuation, connectors, underscores, spaces
   772тЖТ   772тЖТ   772тЖТ * 2. Never split a compound word (e.g., "чобчРЖшзДшМГ" must not become "чобчРЖшзД" + "шМГ")
   773тЖТ   773тЖТ   773тЖТ * 3. No single-character orphan on the last line тАФ merge into previous line
   774тЖТ   774тЖТ   774тЖТ * 4. If no good break point found within 60-130% of charsPerLine, break at charsPerLine
   775тЖТ   775тЖТ   775тЖТ *
   776тЖТ   776тЖТ   776тЖТ * @param {string} title        Full title string
   777тЖТ   777тЖТ   777тЖТ * @param {number} charsPerLine Max characters per line at current font size
   778тЖТ   778тЖТ   778тЖТ * @returns {string[]}          Array of line strings
   779тЖТ   779тЖТ   779тЖТ */
   780тЖТ   780тЖТ   780тЖТfunction splitTitleLines(title, charsPerLine) {
   781тЖТ   781тЖТ   781тЖТ  if (title.length <= charsPerLine) return [title];
   782тЖТ   782тЖТ   782тЖТ
   783тЖТ   783тЖТ   783тЖТ  // Characters that are safe break points (break AFTER these)
   784тЖТ   784тЖТ   784тЖТ  const breakAfter = new Set([
   785тЖТ   785тЖТ   785тЖТ    ...'я╝МуАВуАБя╝Ыя╝Ъя╝Бя╝Я',              // CJK punctuation
   786тЖТ   786тЖТ   786тЖТ    ...'чЪДф╕ОхТМхПКф╣ЛхЬиф║Оф╕║',            // CJK particles/prepositions
   787тЖТ   787тЖТ   787тЖТ    ...'-_тАФтАУ┬╖/',                     // connectors
   788тЖТ   788тЖТ   788тЖТ    ...' \t',                         // whitespace
   789тЖТ   789тЖТ   789тЖТ  ]);
   790тЖТ   790тЖТ   790тЖТ
   791тЖТ   791тЖТ   791тЖТ  const lines = [];
   792тЖТ   792тЖТ   792тЖТ  let remaining = title;
   793тЖТ   793тЖТ   793тЖТ
   794тЖТ   794тЖТ   794тЖТ  while (remaining.length > charsPerLine) {
   795тЖТ   795тЖТ   795тЖТ    let breakAt = -1;
   796тЖТ   796тЖТ   796тЖТ
   797тЖТ   797тЖТ   797тЖТ    // Search backward from charsPerLine to 60% for a break point
   798тЖТ   798тЖТ   798тЖТ    for (let i = charsPerLine; i >= Math.floor(charsPerLine * 0.6); i--) {
   799тЖТ   799тЖТ   799тЖТ      if (i < remaining.length && breakAfter.has(remaining[i - 1])) {
   800тЖТ   800тЖТ   800тЖТ        breakAt = i;
   801тЖТ   801тЖТ   801тЖТ        break;
   802тЖТ   802тЖТ   802тЖТ      }
   803тЖТ   803тЖТ   803тЖТ    }
   804тЖТ   804тЖТ   804тЖТ
   805тЖТ   805тЖТ   805тЖТ    // If not found, search forward up to 130%
   806тЖТ   806тЖТ   806тЖТ    if (breakAt === -1) {
   807тЖТ   807тЖТ   807тЖТ      const limit = Math.min(remaining.length, Math.ceil(charsPerLine * 1.3));
   808тЖТ   808тЖТ   808тЖТ      for (let i = charsPerLine + 1; i < limit; i++) {
   809тЖТ   809тЖТ   809тЖТ        if (breakAfter.has(remaining[i - 1])) {
   810тЖТ   810тЖТ   810тЖТ          breakAt = i;
   811тЖТ   811тЖТ   811тЖТ          break;
   812тЖТ   812тЖТ   812тЖТ        }
   813тЖТ   813тЖТ   813тЖТ      }
   814тЖТ   814тЖТ   814тЖТ    }
   815тЖТ   815тЖТ   815тЖТ
   816тЖТ   816тЖТ   816тЖТ    // Last resort: break at charsPerLine, but avoid splitting compound CJK words
   817тЖТ   817тЖТ   817тЖТ    if (breakAt === -1) {
   818тЖТ   818тЖТ   818тЖТ      breakAt = charsPerLine;
   819тЖТ   819тЖТ   819тЖТ      // If both chars at the break boundary are CJK (likely a compound word),
   820тЖТ   820тЖТ   820тЖТ      // step back 1 char to keep the word together
   821тЖТ   821тЖТ   821тЖТ      const prevChar = remaining[breakAt - 1];
   822тЖТ   822тЖТ   822тЖТ      const nextChar = remaining[breakAt];
   823тЖТ   823тЖТ   823тЖТ      if (prevChar && nextChar &&
   824тЖТ   824тЖТ   824тЖТ          !breakAfter.has(prevChar) && !breakAfter.has(nextChar) &&
   825тЖТ   825тЖТ   825тЖТ          /[\u4e00-\u9fff]/.test(prevChar) && /[\u4e00-\u9fff]/.test(nextChar)) {
   826тЖТ   826тЖТ   826тЖТ        breakAt = breakAt - 1;
   827тЖТ   827тЖТ   827тЖТ      }
   828тЖТ   828тЖТ   828тЖТ    }
   829тЖТ   829тЖТ   829тЖТ
   830тЖТ   830тЖТ   830тЖТ    lines.push(remaining.slice(0, breakAt).trim());
   831тЖТ   831тЖТ   831тЖТ    remaining = remaining.slice(breakAt).trim();
   832тЖТ   832тЖТ   832тЖТ  }
   833тЖТ   833тЖТ   833тЖТ  if (remaining) lines.push(remaining);
   834тЖТ   834тЖТ   834тЖТ
   835тЖТ   835тЖТ   835тЖТ  // Prevent single-character orphan on last line тАФ merge into previous
   836тЖТ   836тЖТ   836тЖТ  if (lines.length > 1 && lines[lines.length - 1].length <= 2) {
   837тЖТ   837тЖТ   837тЖТ    const last = lines.pop();
   838тЖТ   838тЖТ   838тЖТ    lines[lines.length - 1] += last;
   839тЖТ   839тЖТ   839тЖТ  }
   840тЖТ   840тЖТ   840тЖТ
   841тЖТ   841тЖТ   841тЖТ  return lines;
   842тЖТ   842тЖТ   842тЖТ}
   843тЖТ   843тЖТ   843тЖТ
   844тЖТ   844тЖТ   844тЖТ/**
   845тЖТ   845тЖТ   845тЖТ * Calculate dynamic spacing values for cover elements to fit within page height.
   846тЖТ   846тЖТ   846тЖТ *
   847тЖТ   847тЖТ   847тЖТ * @param {object} params
   848тЖТ   848тЖТ   848тЖТ * @param {number} params.titleLineCount   Number of title lines
   849тЖТ   849тЖТ   849тЖТ * @param {number} params.titlePt          Title font size in pt
   850тЖТ   850тЖТ   850тЖТ * @param {boolean} params.hasSubtitle     Whether subtitle exists
   851тЖТ   851тЖТ   851тЖТ * @param {boolean} params.hasEnglishLabel Whether English label exists
   852тЖТ   852тЖТ   852тЖТ * @param {number} params.metaLineCount    Number of meta info lines
   853тЖТ   853тЖТ   853тЖТ * @param {number} params.fixedHeight      Sum of fixed-height elements (color strips, accent bars, footer) in twips
   854тЖТ   854тЖТ   854тЖТ * @param {number} params.pageHeight       Total page height in twips (default 16838)
   855тЖТ   855тЖТ   855тЖТ * @returns {{ topSpacing, midSpacing, bottomSpacing }}  Spacing values in twips
   856тЖТ   856тЖТ   856тЖТ */
   857тЖТ   857тЖТ   857тЖТfunction calcCoverSpacing(params) {
   858тЖТ   858тЖТ   858тЖТ  const {
   859тЖТ   859тЖТ   859тЖТ    titleLineCount = 1, titlePt = 36, hasSubtitle = false,
   860тЖТ   860тЖТ   860тЖТ    hasEnglishLabel = false, metaLineCount = 0,
   861тЖТ   861тЖТ   861тЖТ    fixedHeight = 800, pageHeight = 16838,
   862тЖТ   862тЖТ   862тЖТ    marginTop = 0, marginBottom = 0,   // тШЕ NEW: pass actual section margins
   863тЖТ   863тЖТ   863тЖТ  } = params;
   864тЖТ   864тЖТ   864тЖТ
   865тЖТ   865тЖТ   865тЖТ  // тШЕ Safety margin: 1200 twips (cross-engine: MS Office renders large fonts
   866тЖТ   866тЖТ   866тЖТ  // taller than calculated; extra 400tw buffer prevents footer clipping)
   867тЖТ   867тЖТ   867тЖТ  const SAFETY = 1200;
   868тЖТ   868тЖТ   868тЖТ  // тШЕ Subtract page margins from available height (cover section may have margins)
   869тЖТ   869тЖТ   869тЖТ  const usableHeight = pageHeight - marginTop - marginBottom - SAFETY;
   870тЖТ   870тЖТ   870тЖТ
   871тЖТ   871тЖТ   871тЖТ  // тШЕ Accurate height estimation per element:
   872тЖТ   872тЖТ   872тЖТ  const titleHeight = titleLineCount * (titlePt * 23 + 200);
   873тЖТ   873тЖТ   873тЖТ  const subtitleHeight = hasSubtitle ? (12 * 23 + 600) : 0;
   874тЖТ   874тЖТ   874тЖТ  const englishLabelHeight = hasEnglishLabel ? (9 * 23 + 600) : 0;
   875тЖТ   875тЖТ   875тЖТ  const metaHeight = metaLineCount * (10 * 23 + 100);
   876тЖТ   876тЖТ   876тЖТ
   877тЖТ   877тЖТ   877тЖТ  // тШЕ Account for implicit paragraph heights:
   878тЖТ   878тЖТ   878тЖТ  const implicitParaHeight = 3 * 300;
   879тЖТ   879тЖТ   879тЖТ
   880тЖТ   880тЖТ   880тЖТ  const contentHeight = titleHeight + subtitleHeight + englishLabelHeight +
   881тЖТ   881тЖТ   881тЖТ                        metaHeight + fixedHeight + implicitParaHeight;
   882тЖТ   882тЖТ   882тЖТ
   883тЖТ   883тЖТ   883тЖТ  const remainingSpace = usableHeight - contentHeight;
   884тЖТ   884тЖТ   884тЖТ  const safeRemaining = Math.max(remainingSpace, 400);
   885тЖТ   885тЖТ   885тЖТ
   886тЖТ   886тЖТ   886тЖТ  // тШЕ Footer protection: bottomSpacing must be тЙе FOOTER_MIN to prevent
   887тЖТ   887тЖТ   887тЖТ  // footer + accent line from being clipped at the cell bottom edge
   888тЖТ   888тЖТ   888тЖТ  const FOOTER_MIN = 800;
   889тЖТ   889тЖТ   889тЖТ  const rawTop = Math.floor(safeRemaining * 0.45);
   890тЖТ   890тЖТ   890тЖТ  const rawBottom = Math.floor(safeRemaining * 0.45);
   891тЖТ   891тЖТ   891тЖТ  const bottomSpacing = Math.max(rawBottom, FOOTER_MIN);
   892тЖТ   892тЖТ   892тЖТ  const topSpacing = Math.max(rawTop - Math.max(0, FOOTER_MIN - rawBottom), 400);
   893тЖТ   893тЖТ   893тЖТ  const midSpacing = Math.max(safeRemaining - topSpacing - bottomSpacing, 0);
   894тЖТ   894тЖТ   894тЖТ
   895тЖТ   895тЖТ   895тЖТ  return { topSpacing, midSpacing, bottomSpacing };
   896тЖТ   896тЖТ   896тЖТ}
   897тЖТ   897тЖТ   897тЖТ```
   898тЖТ   898тЖТ   898тЖТ
   899тЖТ   899тЖТ   899тЖТ**Usage in every recipe:**
   900тЖТ   900тЖТ   900тЖТ```js

   926тЖТ   926тЖТ   926тЖТ### тЪая╕П CRITICAL тАФ Cover Section Non-Negotiables (ALL Recipes)
   927тЖТ   927тЖТ   927тЖТ
   928тЖТ   928тЖТ   928тЖТThese 3 properties are MANDATORY for every cover implementation (R1тАУR7). Omitting ANY of them causes cover layout failure:
   929тЖТ   929тЖТ   929тЖТ
   930тЖТ   930тЖТ   930тЖТ1. **Cover section margin = 0**: The cover MUST be in its own section with `page.margin: { top: 0, bottom: 0, left: 0, right: 0 }`. Non-zero margins shrink the wrapper away from page edges тЖТ white gaps around the cover ("cover not filling the page"). This is the #1 cause of broken cover layouts.
   931тЖТ   931тЖТ   931тЖТ
   932тЖТ   932тЖТ   932тЖТ2. **Wrapper row exact height**: The outer wrapper table row MUST set `height: { value: 16838, rule: "exact" }`. Without this, content overflow pushes to page 2, or insufficient content leaves bottom whitespace.
   933тЖТ   933тЖТ   933тЖТ
   934тЖТ   934тЖТ   934тЖТ3. **Wrapper table borders = allNoBorders**: MUST explicitly set `borders: allNoBorders`. Default docx-js borders add ~8 twips per edge.   1445тЖТ  1445тЖТ  1445тЖТ### Recipe R5: Clean White (Academic)
  1446тЖТ  1446тЖТ  1446тЖТ
  1447тЖТ  1447тЖТ  1447тЖТ**Visual:** Pure white background + school name + centered title + 2-column meta info table with underlines + footer
  1448тЖТ  1448тЖТ  1448тЖТ**Use case:** academic thesis, standards documents
  1449тЖТ  1449тЖТ  1449тЖТ**Architecture:** 16838 outer wrapper (white fill, invisible) + cell margins for page margin simulation. No top/bottom decorative lines.
  1450тЖТ  1450тЖТ  1450тЖТ
  1451тЖТ  1451тЖТ  1451тЖТ**Meta info table rules (cross-engine safe):**
  1452тЖТ  1452тЖТ  1452тЖТ- 2-column table: label + value, **percentage widths only** (`WidthType.PERCENTAGE`)
  1453тЖТ  1453тЖТ  1453тЖТ- **Table width is adaptive:** 55тАУ75% of page, calculated by `calcR5MetaLayout()`. Table centered via `alignment: CENTER`.
  1454тЖТ  1454тЖТ  1454тЖТ- **Label column:** adaptive 25тАУ45% of table, **LEFT aligned**, plain text with "я╝Ъ" appended. NO full-width space padding, NO right-alignment.
  1455тЖТ  1455тЖТ  1455тЖТ- **Label column borders:** none (no bottom border on label cells).
  1456тЖТ  1456тЖТ  1456тЖТ- **Value column:** remaining %, **LEFT aligned**, `bottom: single sz=4` border = fixed-length underline (consistent length for all rows regardless of text).
  1457тЖТ  1457тЖТ  1457тЖТ- No left/right/top borders on either column.
  1458тЖТ  1458тЖТ  1458тЖТ- тЪая╕П Do NOT use DXA widths, full-width space padding (`\u3000`), spacer columns, or tab stops тАФ WPS renders them inconsistently.
  1459тЖТ  1459тЖТ  1459тЖТ- тЪая╕П Do NOT use `margins.top` on the wrapper cell тАФ use `spacing.before` on first paragraph instead.
  1460тЖТ  1460тЖТ  1460тЖТ
  1461тЖТ  1461тЖТ  1461тЖТ**Known limitation:** When meta lines тЙе 6 AND title has 3 lines, MS Office may render content slightly taller than WPS, potentially clipping the footer line. Mitigate by reducing `midSpacing` or using a smaller title font.
  1462тЖТ  1462тЖТ  1462тЖТ
  1463тЖТ  1463тЖТ  1463тЖТ```js
  1464тЖТ  1464тЖТ  1464тЖТ// тФАтФА Width-aware title layout (handles mixed Chinese + English) тФАтФА
  1465тЖТ  1465тЖТ  1465тЖТfunction estimateTextWidth(text, pt) {
  1466тЖТ  1466тЖТ  1466тЖТ  let width = 0;
  1467тЖТ  1467тЖТ  1467тЖТ  for (const ch of text) {
  1468тЖТ  1468тЖТ  1468тЖТ    const code = ch.codePointAt(0);
  1469тЖТ  1469тЖТ  1469тЖТ    const isCJK = (code >= 0x4E00 && code <= 0x9FFF) || (code >= 0x3400 && code <= 0x4DBF) ||
  1470тЖТ  1470тЖТ  1470тЖТ      (code >= 0x3000 && code <= 0x303F) || (code >= 0xFF00 && code <= 0xFFEF) ||
  1471тЖТ  1471тЖТ  1471тЖТ      (code >= 0x2E80 && code <= 0x2EFF);
  1472тЖТ  1472тЖТ  1472тЖТ    width += isCJK ? pt * 20 : pt * 11; // CJK: full-width, Latin: ~55% width
  1473тЖТ  1473тЖТ  1473тЖТ  }
  1474тЖТ  1474тЖТ  1474тЖТ  return width;
  1475тЖТ  1475тЖТ  1475тЖТ}
  1476тЖТ  1476тЖТ  1476тЖТ// Use estimateTextWidth() in calcTitleLayout() instead of simple char count
  1477тЖТ  1477тЖТ  1477тЖТ// to prevent mid-word breaks in mixed Chinese+English titles like "хЯ║ф║ОTransformerцЮ╢цЮДчЪД..."
  1478тЖТ  1478тЖТ  1478тЖТ
  1479тЖТ  1479тЖТ  1479тЖТ// тФАтФА Meta info table тФАтФА
  1480тЖТ  1480тЖТ  1480тЖТ
  1481тЖТ  1481тЖТ  1481тЖТ// Calculate adaptive table and label column percentage based on longest label.
  1482тЖТ  1482тЖТ  1482тЖТ// Returns { tablePct, labelPct } тАФ both as percentages.
  1483тЖТ  1483тЖТ  1483тЖТ// Uses ONLY WidthType.PERCENTAGE for cross-engine compatibility (MS Office + WPS).
  1484тЖТ  1484тЖТ  1484тЖТfunction calcR5MetaLayout(metaEntries, fontPt = 12) {
  1485тЖТ  1485тЖТ  1485тЖТ  const maxLabelLen = Math.max(...metaEntries.map(e => [...e.label].length));
  1486тЖТ  1486тЖТ  1486тЖТ  // Label needs: maxLabelLen chars + "я╝Ъ" + 1 char padding
  1487тЖТ  1487тЖТ  1487тЖТ  const labelNeedTw = (maxLabelLen + 2) * fontPt * 20;
  1488тЖТ  1488тЖТ  1488тЖТ  // Value column: fixed ~5000tw for consistent underline length
  1489тЖТ  1489тЖТ  1489тЖТ  const valueNeedTw = 5000;
  1490тЖТ  1490тЖТ  1490тЖТ  const totalNeedTw = labelNeedTw + valueNeedTw;
  1491тЖТ  1491тЖТ  1491тЖТ  // Table width as % of page (11906tw), clamped to 55тАУ75%
  1492тЖТ  1492тЖТ  1492тЖТ  const tablePct = Math.min(75, Math.max(55, Math.ceil(totalNeedTw / 11906 * 100)));
  1493тЖТ  1493тЖТ  1493тЖТ  // Label % within the table, clamped to 25тАУ45%
  1494тЖТ  1494тЖТ  1494тЖТ  const rawLabelPct = Math.ceil(labelNeedTw / (tablePct / 100 * 11906) * 100);
  1495тЖТ  1495тЖТ  1495тЖТ  return { tablePct, labelPct: Math.max(25, Math.min(45, rawLabelPct)) };
  1496тЖТ  1496тЖТ  1496тЖТ}
  1497тЖТ  1497тЖТ  1497тЖТ
  1498тЖТ  1498тЖТ  1498тЖТ// Build R5 academic cover meta info table.
  1499тЖТ  1499тЖТ  1499тЖТ// тЪая╕П CRITICAL cross-engine rules:
  1500тЖТ  1500тЖТ  1500тЖТ//   - Table width: WidthType.PERCENTAGE (NOT DXA тАФ WPS breaks with DXA)
  1501тЖТ  1501тЖТ  1501тЖТ//   - Column widths: WidthType.PERCENTAGE
  1502тЖТ  1502тЖТ  1502тЖТ//   - Label column: LEFT aligned, plain text (NO full-width space padding)
  1503тЖТ  1503тЖТ  1503тЖТ//   - Value column: LEFT aligned, bottom border = fixed-length underline
  1504тЖТ  1504тЖТ  1504тЖТ//   - Table alignment: CENTER (visually centered on page)

  1549тЖТ  1549тЖТ  1549тЖТ    rows,
  1550тЖТ  1550тЖТ  1550тЖТ  });
  1551тЖТ  1551тЖТ  1551тЖТ}
  1552тЖТ  1552тЖТ  1552тЖТ
  1553тЖТ  1553тЖТ  1553тЖТ// тЪая╕П MANDATORY: Cover section must use margin: 0. See "Cover Section Non-Negotiables" above.
  1554тЖТ  1554тЖТ  1554тЖТfunction buildCoverR5(config) {
  1555тЖТ  1555тЖТ  1555тЖТ  const PAGE_H = 16838, SAFETY = 1200;

  1562тЖТ  1562тЖТ  1562тЖТ  const titleSize = titlePt * 2;
  1563тЖТ  1563тЖТ  1563тЖТ

  1657тЖТ  1657тЖТ  1657тЖТ### blendColors Utility Function
  1658тЖТ  1658тЖТ  1658тЖТ
  1659тЖТ  1659тЖТ  1659тЖТ```js
  1660тЖТ  1660тЖТ  1660тЖТfunction blendColors(hex1, hex2, ratio) {
  1661тЖТ  1661тЖТ  1661тЖТ  const p = (s, i) => parseInt(s.replace("#","").slice(i, i+2), 16);
  1662тЖТ  1662тЖТ  1662тЖТ  const mix = (c1, c2) => Math.round(c1 + (c2 - c1) * ratio);
  1663тЖТ  1663тЖТ  1663тЖТ  const r = mix(p(hex1,0), p(hex2,0)), g = mix(p(hex1,2), p(hex2,2)), b = mix(p(hex1,4), p(hex2,4));
  1664тЖТ  1664тЖТ  1664тЖТ  return [r, g, b].map(v => v.toString(16).padStart(2,"0")).join("");
  1665тЖТ  1665тЖТ  1665тЖТ}
  1666тЖТ  1666тЖТ  1666тЖТ```

  1717тЖТ  1717тЖТ  1717тЖТ**Color usage rules:**
  1718тЖТ  1718тЖТ  1718тЖТ- Max **5 colors** per chart; excess categories use depth variants of same hue
  1719тЖТ  1719тЖТ  1719тЖТ- Emphasis data uses document accent color; non-emphasis uses grey `#B0B0B0`
  1720тЖТ  1720тЖТ  1720тЖТ- Adjacent segments in pie/bar charts must have hue gap тЙе 60┬░
  1721тЖТ  1721тЖТ  1721тЖТ
  1722тЖТ  1722тЖТ  1722тЖТ1. **Anti-overlap**: If >6 x-axis labels, rotate 45┬░ (`plt.xticks(rotation=45, ha='right')`)
  1723тЖТ  1723тЖТ  1723тЖТ2. **Anti-stretch**: Always set figure size explicitly (`fig, ax = plt.subplots(figsize=(10, 6))`)
  1724тЖТ  1724тЖТ  1724тЖТ3. **Aspect ratio (CRITICAL)**: When embedding in docx, MUST read actual image dimensions and calculate height proportionally. NEVER hardcode both width and height тАФ pie charts become ellipses, radar charts become diamonds.
  1725тЖТ  1725тЖТ  1725тЖТ   ```js
  1726тЖТ  1726тЖТ  1726тЖТ   const sizeOf = require("image-size");
  1727тЖТ  1727тЖТ  1727тЖТ   const dims = sizeOf(chartBuffer);
  1728тЖТ  1728тЖТ  1728тЖТ   const displayWidth = 500;
  1729тЖТ  1729тЖТ  1729тЖТ   const displayHeight = Math.round(displayWidth * (dims.height / dims.width));
  1730тЖТ  1730тЖТ  1730тЖТ   // transformation: { width: displayWidth, height: displayHeight }
  1731тЖТ  1731тЖТ  1731тЖТ   ```
  1732тЖТ  1732тЖТ  1732тЖТ4. **DPI**: Save at 200+ DPI (`plt.savefig("chart.png", dpi=200, bbox_inches="tight")`)
  1733тЖТ  1733тЖТ  1733тЖТ5. **Colors**: Use palette accent color for primary data series
  1734тЖТ  1734тЖТ  1734тЖТ6. **Legend**: Place outside plot area if >4 series
  1735тЖТ  1735тЖТ  1735тЖТ7. **Square charts**: Pie and radar charts MUST use `figsize=(8, 8)` (equal width/height) to preserve circular/radial shape
  1736тЖТ  1736тЖТ  1736тЖТ7. **Grid**: Light gray grid (`ax.grid(True, alpha=0.3)`)
